### 💡 遇到的问题与备忘 (Issues & Notes)

#### 1. Qlib 回测 `-1` 错误与工件加载
- **问题**: 回测服务无法通过 Recorder 加载 `pred.pkl`，导致回测失败。
- **原因**: Qlib 内部路径解析问题或 Recorder 状态不一致。
- **解决**: 修改 `train_model` 返回明确的 `pred_path`（绝对路径），并在 `run_backtest` 中优先使用该路径直接加载 `pd.read_pickle`，绕过 Recorder 的加载机制。

#### 2. Dataset 类不匹配 (`Dataset` vs `DatasetH`)
- **问题**: 日志报错 `'Dataset' object has no attribute 'segments'`。
- **原因**: 代码使用了基础 `Dataset` 类，而 Qlib 的 `run_workflow` 可以在配置中指定 `DatasetH`，但手动实例化时需要显式使用 `DatasetH` 类以支持数据分片。
- **解决**: 在 `qlib_workflow_service.py` 中引入并使用 `DatasetH` 替代 `Dataset`。

#### 3. Windows 环境下的多进程崩溃 (Process Crash)
- **问题**: 修复 Dataset 后，训练任务在 API 调用（结合 `TestClient`）时静默失败，无日志输出。
- **原因**: 即使在配置文件中尝试禁用多进程，Windows 下的 `multiprocessing` 在复杂应用（FastAPI + Qlib）中仍可能产生冲突或资源锁死。
- **解决 (Critical)**: **进程隔离模式**。
    - 创建独立脚本 `src/zenith_ai/services/qlib_train_worker.py` 封装训练逻辑。
    - 使用 `subprocess.run` 在全新进程中执行训练，彻底规避主进程环境干扰。
    - 通过 stdout 捕获 JSON 格式的训练结果。

#### 4. 依赖项管理
- `httpx` 需要锁定在 `<0.28.0` 以兼容 `starlette` 的 `TestClient`。
- `catboost` 是 Qlib `LGBModel` 或其他部分依赖的组件，需显式安装。
